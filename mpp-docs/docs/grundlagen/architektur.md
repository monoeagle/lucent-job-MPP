# Architektur

MPP folgt **Clean Architecture** auf beiden Seiten (Backend und Frontend) mit strikter Dependency-Richtung.

---

## Prinzip

Abhaengigkeiten zeigen immer nach innen вҖ” von der Infrastruktur zum Domaenkern. Die aeusseren Schichten kennen die inneren, aber nicht umgekehrt.

### Dependency-Flow

```
api/ вҶ’ services/ вҶ’ domain/ вҶҗ data/
```

- `api/` darf `services/` und `core/` aufrufen
- `services/` darf `domain/` und `data/repositories/` aufrufen
- `data/` darf `domain/` aufrufen
- `domain/` hat keine Abhaengigkeiten

### Verboten

- `api/` вҶ’ `data/` (nur ueber Services)
- `domain/` вҶ’ `data/`, `services/`, `api/`
- `services/` вҶ’ `api/`
