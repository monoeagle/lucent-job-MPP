---
name: User - Rolle und Kontext
description: User ist Product Owner / technischer Stakeholder für ein IT-Services Marketplace Portal
type: user
---

Der User agiert als technischer Stakeholder / Auftraggeber für ein Marketplace Portal für automatisierte IT-Services. Er erstellt Feature-Spezifikationen im definierten PO-Format (User Story, REQ, VAL, API Contract, Edge Cases).

Der User arbeitet mit einem Entwicklerteam, das die Specs ohne Rückfragen umsetzen soll — daher ist Vollständigkeit und Eindeutigkeit wichtig.

Kontext: IT-Infrastruktur-Marketplace (intern oder B2B), keine klassische Consumer-Plattform.
